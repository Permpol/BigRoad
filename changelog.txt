===============================================================================
  XAUUSD Big Road - Baccarat Style : รายละเอียดการแก้ไข
===============================================================================

1. แก้ไขการ Plot ตามผลการทำนายของ User (Win/Loss) ไม่ใช่ผลจริง
-------------------------------------------------------------------------------
   - เดิม: plot ตามผลจริงของแท่งเทียน (bull หรือ bear ตาม close > open)
   - ใหม่: plot ตามผลการทำนายของ user
     * ถ้า user เลือกตรงกับผลจริง (WIN) → plot BULL (วงกลมน้ำเงิน ▲)
     * ถ้า user เลือกไม่ตรงกับผลจริง (LOSS) → plot BEAR (วงกลมแดง ▼)
   - เหตุผล: ต้องการให้ Big Road แสดงผลสถิติการทำนายของ user
     ว่าชนะหรือแพ้ต่อเนื่องกี่ครั้ง เพื่อวิเคราะห์ pattern การตัดสินใจ
   - Stats เปลี่ยนจาก BULL/BEAR เป็น WIN/LOSS

2. เปลี่ยนเวลา Betting จาก ~2 วินาที เป็น 2 นาที (กรอกค่าเองได้)
-------------------------------------------------------------------------------
   - เดิม: betting phase = 40% ของ 5 วินาที = ~2 วินาที (สั้นมาก)
   - ใหม่: เพิ่ม input "BET TIME (MIN)" ค่าเริ่มต้น = 2 นาที
     * user สามารถกรอกค่าเองได้ (0.5 - 30 นาที)
     * ถ้ากรอกค่าใหม่ขณะเกมกำลังรัน → เกมจะ restart ใหม่โดยอัตโนมัติ
   - เหตุผล: เวลา 2 วินาทีเดิมสั้นเกินไป ทำให้ user ตัดสินใจไม่ทัน
     2 นาทีให้เวลาวิเคราะห์ราคาก่อนวางเดิมพัน

3. เปลี่ยนเวลา Background (ช่วงรอ) จาก ~3 วินาที เป็น 10 นาที (กรอกค่าเองได้)
-------------------------------------------------------------------------------
   - เดิม: background/waiting phase = 60% ของ 5 วินาที = ~3 วินาที
   - ใหม่: เพิ่ม input "BG TIME (MIN)" ค่าเริ่มต้น = 10 นาที
     * user สามารถกรอกค่าเองได้ (1 - 60 นาที)
     * ถ้ากรอกค่าใหม่ขณะเกมกำลังรัน → เกมจะ restart ใหม่โดยอัตโนมัติ
   - เหตุผล: ช่วง background ใช้สำหรับแสดงราคาแท่งเทียนปัจจุบัน
     10 นาทีให้เวลา user ดูการเคลื่อนไหวของราคาก่อนเข้าสู่ช่วง betting
   - Timer แสดงเวลาแบบ mm:ss เพื่อรองรับเวลาที่ยาวขึ้น

4. แก้ไข Big Road ให้รองรับรูปแบบ L-pattern (Dragon Tail)
-------------------------------------------------------------------------------
   - เดิม: เมื่อ column เต็ม 6 แถว แล้วมีผลเดียวกันมาอีก → เริ่ม column ใหม่
     จากแถวที่ 1 (ไม่ถูกต้องตามหลัก Big Road)
   - ใหม่: ใช้ระบบ grid-based (col, row) tracking
     * ผลเดียวกัน: ลงล่างในคอลัมน์เดิม
     * ถ้าคอลัมน์เต็ม 6 แถว + ผลเดียวกัน: ขยายไปทางขวาที่แถวเดิม
       (สร้างรูปตัว L / Dragon Tail)
     * ผลต่างกัน: เริ่มคอลัมน์ใหม่จากแถวที่ 1
   - เหตุผล: ตามหลักการบันทึก Big Road มาตรฐานในบาคาร่า
     L-pattern (Dragon Tail) แสดงให้เห็น streak ยาวๆ ได้ชัดเจน
     ช่วยให้วิเคราะห์ pattern การชนะ/แพ้ต่อเนื่องได้ดีขึ้น

5. เพิ่ม input "CANDLE (MIN)" ให้ restart เกมเมื่อเปลี่ยนค่า
-------------------------------------------------------------------------------
   - เดิม: เปลี่ยนค่า candle duration แล้วต้อง reload หน้าเว็บ
   - ใหม่: เปลี่ยนค่า → build candles ใหม่ + restart เกมอัตโนมัติ
   - เหตุผล: ให้ user ทดลองค่าต่างๆ ได้สะดวกโดยไม่ต้อง reload

6. แก้ไข Flow เวลา: BET เปิดก่อน → รอดูราคา → ตรวจผล
-------------------------------------------------------------------------------
   - เดิม: [BG รอดูราคา] → [BET วางเดิมพัน] → ผล (BET + BG แยกกัน)
   - ใหม่: [BET วางเดิมพันทันที] → [WAITING รอดูราคา] → ผล
     * BET TIME รวมอยู่ใน BG TIME (ไม่บวกเพิ่ม)
     * รอบเริ่ม → เปิด BET ทันที (2 นาที)
     * BET ปิด → เข้าสู่ WAITING ดูราคา (อีก 8 นาทีที่เหลือ)
     * ครบ BG TIME (10 นาที) → ตรวจผล
   - เหตุผล: ให้ user ตัดสินใจก่อน แล้วรอดูว่าผลตรงกับที่ทำนายหรือไม่
     สอดคล้องกับหลักการพนันที่ต้องวางเดิมพันก่อนเปิดไพ่

7. เพิ่มฟังก์ชัน Speed สำหรับ Backtest
-------------------------------------------------------------------------------
   - เพิ่ม dropdown "SPEED" ให้เลือกได้หลายระดับ:
     * 1x (ปกติ), 2x, 5x, 10x, 30x, 60x, 300x (Backtest), MAX (ข้ามเวลา)
   - Speed หารเวลาทุกอย่าง: BET TIME, BG TIME, pause ระหว่างรอบ
     เช่น BG=10 นาที ที่ 60x → เหลือ 10 วินาทีต่อรอบ
   - MAX speed: ข้ามเวลาทั้งหมด รันผลทันที (สำหรับ backtest เร็วสุด)
   - เปลี่ยน speed ระหว่างเกม → เริ่มรอบปัจจุบันใหม่ด้วย speed ใหม่ทันที
   - tick rate ปรับตามความเร็ว เพื่อให้ animation ลื่นที่ทุกระดับ
   - เหตุผล: ให้ user ทดสอบกลยุทธ์ได้เร็วขึ้นโดยไม่ต้องรอเวลาจริง

8. แปลง Vercel Project + ระบบป้องกันการ Copy
-------------------------------------------------------------------------------
   โครงสร้าง:
     vercel-project/
     ├── pages/
     │   ├── index.js          ← หน้า Login (ใส่ password)
     │   ├── game.js           ← หน้า Big Road (protected)
     │   └── api/
     │       ├── auth.js       ← ตรวจ password → ออก JWT token (HttpOnly cookie)
     │       └── data.js       ← serve CSV เป็น JSON (protected, ไม่ให้ download)
     ├── middleware.js          ← block /game + /api/data ถ้าไม่มี token
     ├── lib/jwt.js             ← JWT sign/verify (jose library)
     ├── styles/game.css        ← CSS แยกไฟล์
     ├── public/                ← ว่างเปล่า! (ไม่วาง data.csv ที่นี่เด็ดขาด)
     ├── data/data.csv          ← อยู่ใน server เท่านั้น (152,905 ticks)
     ├── next.config.js         ← CSP headers, ปิด source map, ปิด X-Powered-By
     ├── .env.example           ← ตัวอย่าง APP_PASSWORD + JWT_SECRET
     └── package.json           ← Next.js 14 + jose

   ระบบป้องกัน (หลายชั้น):
     1. JWT Authentication    → ต้อง login ถูกก่อนถึงจะเข้า /game ได้
     2. HttpOnly Cookie       → JS อ่าน token ไม่ได้
     3. Edge Middleware        → block ทุก request ไปยัง protected routes
     4. CSV ไม่ serve ตรง     → /api/data แปลงเป็น JSON แล้วส่ง (no file download)
     5. No-store headers      → ห้าม cache, ห้าม download
     6. Disable right-click   → contextmenu blocked
     7. Disable shortcuts     → F12, Ctrl+Shift+I/J/C, Ctrl+U/S/P blocked
     8. Disable drag          → ลากข้อมูลไม่ได้
     9. Disable select        → CSS user-select: none + @media print hidden
    10. Debugger trap          → ตรวจ devtools ด้วย timing (ถ้าเปิด → หน้าว่าง)
    11. No source maps         → productionBrowserSourceMaps: false
    12. CSP headers            → Content-Security-Policy ป้องกัน XSS/injection
    13. X-Frame-Options: DENY  → ไม่ให้ iframe embed

   วิธี Deploy:
     1. cd vercel-project
     2. cp .env.example .env.local  แล้วแก้ค่า PASSWORD + SECRET
     3. npm install
     4. npm run dev            (ทดสอบ local)
     5. vercel deploy          (deploy ขึ้น Vercel)

===============================================================================
  สรุปโครงสร้างเวลาใหม่ต่อ 1 รอบ (BG = 10 นาที, BET = 2 นาที):
  [BET: 2 นาที (วางเดิมพัน)] → [WAIT: 8 นาที (ดูราคา)] → ตรวจผล
  รวมทั้งหมด = BG TIME = 10 นาที (BET TIME อยู่ข้างใน)
